# Тесты для Dear Dates

Этот каталог содержит unit-тесты и UI-тесты для приложения Dear Dates.

## Структура тестов

### Unit-тесты

1. **ProfileTests.swift** - Тесты для модели Profile
   - Вычисление возраста
   - Вычисление следующего дня рождения
   - Вычисление дней до дня рождения
   - Проверка, является ли сегодня днем рождения
   - Обработка високосных годов

2. **DataManagerTests.swift** - Тесты для DataManager
   - CRUD операции для профилей
   - CRUD операции для подарков
   - Сортировка профилей по дням рождения
   - Фильтрация подарков (идеи vs подаренные)
   - Статистика

3. **DataMigrationTests.swift** - Тесты миграции данных
   - Обратная совместимость Profile (avatarColorHue, isFavorite)
   - Миграция AppSettings (isDarkMode → themeType)
   - Кодирование/декодирование данных

4. **NotificationManagerTests.swift** - Тесты для NotificationManager
   - Запрос разрешений
   - Планирование уведомлений
   - Отмена уведомлений
   - Обновление уведомлений

### UI-тесты

5. **DearDatesUITests.swift** - UI-тесты для основных сценариев
   - Онбординг
   - Создание профилей
   - Навигация между вкладками
   - Поиск
   - Настройки
   - Обработка ошибок

## Запуск тестов

### В Xcode

1. Откройте проект в Xcode
2. Нажмите `Cmd + U` для запуска всех тестов
3. Или выберите конкретный тест-файл и нажмите кнопку "Run" рядом с тестом

### Через командную строку

```bash
# Запуск всех тестов
xcodebuild test -scheme "Dear Dates" -destination 'platform=iOS Simulator,name=iPhone 15'

# Запуск конкретного теста
xcodebuild test -scheme "Dear Dates" -destination 'platform=iOS Simulator,name=iPhone 15' -only-testing:DearDatesTests/ProfileTests
```

## Настройка Test Target

Для работы тестов необходимо:

1. Создать Test Target в Xcode (если еще не создан):
   - File → New → Target
   - Выбрать "iOS Unit Testing Bundle" или "iOS UI Testing Bundle"
   - Назвать "Dear Dates Tests"

2. Добавить файлы тестов в Test Target:
   - Выбрать файлы тестов
   - В File Inspector отметить "Dear Dates Tests" в Target Membership

3. Убедиться, что `@testable import Dear_Dates` работает:
   - Проверить, что имя модуля приложения совпадает с именем в import
